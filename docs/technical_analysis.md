# 專案技術架構與限制分析報告

本專案已成功從 Legacy Python 系統遷移為基於 Vite 的高效能 PWA (Progressive Web App)，旨在提供跨平台的報表自動化體驗。

## 1. 核心技術棧 (Technology Stack)

### 前端框架與開發工具
- **React 19 & TypeScript**: 提供強型別的開發環境，確保報表邏輯（如加總、日期運算）的精準度。
- **Vite**: 極速的建置工具，支撐 PWA 的快速熱更新與推播。
- **Tailwind CSS**: 現代化樣式框架，確保介面在手機與電腦端都能展現高質感的「翠綠色」專業工務風格。

### 數據庫與持久化
- **Zustand & IndexedDB**: 使用 Zustand 進行狀態管理，並透過 `idb-keyval` 將所有員工資料、車輛紀錄與國定假日規則永久儲存在瀏覽器的 `IndexedDB` 中。
- **特性**：資料在重新整理網頁後不會消失，且具備離線讀取能力。

### Excel 處理引擎
- **ExcelJS (核心引擎)**：
    - 負責讀取 `.xlsx` 模板。
    - **標籤替換**：自動將 `{{name}}` 等標籤替換為真實資料。
    - **邊框修補演算法**：本專案開發了獨門的「聯集邊框修復程序」，解決了瀏覽器生成 Excel 時常見的框線斷裂與合併儲存格樣式丟失問題。

### PWA 部署與離線
- **vite-plugin-pwa**:
    - **Service Worker**: 自動快取所有 JS、樣板與圖示。
    - **Prompt 模式**：發佈新版時會主動跳出更新提示。
    - **離線支援**：支援 100% 離線產出報表。

## 2. 支援平台 (Platforms)

本系統由於採用 PWA 技術，具備「一次開發，到處運行」的特性：

- **Windows / macOS**:
    - 支援所有現代瀏覽器 (Chrome, Edge, Safari)。
    - 可「安裝」到電腦桌面，像一般軟體一樣開啟。
- **Android / iOS**:
    - 透過 Chrome 或 Safari「加入主畫面」。
    - 針對行動端進行了響應式佈局優化。
- **部署平台**:
    - 目前部署於 **GitHub Pages**。

---

## 3. 使用限制與約束 (Usage Constraints)

雖然 PWA 非常強大，但目前的純前端架構仍有以下限制：

### 數據同步限制
- > [!WARNING]
  > **資料隨瀏覽器走**：目前的資料是存放在「本地瀏覽器」中。如果您在公司電腦輸入資料，回家開個人電腦是看不到的（除非未來加入雲端數據庫同步功能）。
- **數據備份**：若使用者「強制清理瀏覽器快取與資料」，且沒有匯出備份，數據將會永久消失。

### 性能限制
- **大數據負荷**：ExcelJS 在處理極大檔案（例如上千個分頁）時，會消耗較多裝置記憶體，這在低階手機上可能會導致閃退。
- **模板依賴**：系統高度依賴 `public/templates/` 底下的樣板格式。若樣板結構被毀滅性修改，程式碼的標籤定位可能需要同步調整。

### 離線限制
- **首次加載**：第一次開啟網頁必須有網路，以完成 Service Worker 的快取下載。之後才能進入 100% 離線模式。

## 4. 未來擴展方向 (Potential Enhancements)
- **雲端同步**：串接 Google Sheets API 或 Firebase，達成手機電腦資料同步。

- **PDF 匯出**：目前僅限 Excel，未來可增加直接轉 PDF 功能。
